Bối Cảnh & Đặt Vấn Đề

01 / Vấn Đề
Hạn Chế Của LLM Chatbot
Chatbot thông thường hoạt động theo phản xạ nhanh (System 1). Khi được yêu cầu thiết kế đề thi hoặc quản lý ngân hàng câu hỏi:
  • Dễ bị ảo tưởng kiến thức (Hallucination), tự bịa ra đề thi.
  • Không có khả năng tương tác với cơ sở dữ liệu thực.
  • Không thể lưu lại câu hỏi giáo viên thiết kế mới.
Giải Pháp: ReAct Agent
Ứng dụng mô hình tư duy chậm (System 2) kết hợp lập luận và hành động (Reasoning + Acting):
  • Bắt buộc suy nghĩ Thought từng bước rõ ràng.
  • Chủ động gọi các công cụ thực tế Action.
  • Tự sửa sai thông qua kết quả phản hồi từ môi trường Observation.

Kiến Trúc ReAct Loop

02 / Kiến trúc
Quy Trình Tương Tác Các Tác Tử
Teacher ⇒ Yêu cầu tạo đề thi Toán 12 (1 Dễ, 2 Trung bình)
Agent ⇒ Thought: Tra cứu chương trình học Toán lớp 12
Agent ⇒ Action: get_curriculum_topics(12, "Toán")
Tools ⇒ Observation: ["Hàm số lũy thừa", "Khối đa diện"...]
Agent ⇒ Thought: Tìm câu hỏi. Gọi fetch_questions_from_bank(...)
Agent ⇒ Action: save_question_to_bank(...) (Tự sinh khi thiếu)
Agent ⇒ Final Answer: Đề thi hoàn chỉnh kèm LaTeX & đáp án
Cơ Chế Dừng (Stop Sequence)
Sử dụng cấu hình stop=["Observation:", "observation:"].

Giúp ngắt dòng sinh của LLM ngay khi vừa xuất lệnh gọi Tool, chặn đứng lỗi ảo tưởng kết quả công cụ và nhường quyền kiểm soát tuyệt đối cho Python loop thực thi.

Kho Công Cụ Hỗ Trợ (Tools)

03 / Công cụ
Tên Công Cụ Đầu Vào (Arguments) Chức Năng Chính / Vai Trò
get_curriculum_topics grade (int), subject (str) Tra cứu danh sách chủ đề chính thức theo chương trình Bộ GD&ĐT (Toán, Lý lớp 10, 11, 12). Bảo đảm Agent chọn đúng trọng tâm.
fetch_questions_from_bank topic (str), difficulty (str), num_questions (int) Truy vấn ngân hàng câu hỏi có sẵn từ tệp tin JSON cơ sở dữ liệu. Tiết kiệm tài nguyên và giữ tính nhất quán.
save_question_to_bank question_text, options, correct_answer, explanation, difficulty, topic... Tự động ghi nhận câu hỏi mới thiết kế chuẩn sư phạm vào ngân hàng lưu trữ dữ liệu để sử dụng cho lần sau.

Giải Pháp Công Nghệ Đột Phá

04 / Giải pháp
Thuật Toán Khớp Ngoặc Động (Character Scan)
Hàm _find_action_call tự phát triển thay thế Regex cổ điển:
  • Duyệt từng ký tự, ghi nhớ trạng thái chuỗi đóng/mở nháy.
  • Bỏ qua các ký tự ngoặc toán học lồng nhau bên trong biểu thức LaTeX như: y = (x^2 - 4)^0.5.
  • Tránh 100% các lỗi trích xuất thiếu tham số gây crash Agent.
Tương Thích Mọi Endpoint & Fallback DeepSeek
Bộ điều khiển openai_provider.py:
  • Hỗ trợ đổi endpoint động qua cấu hình LLM_ENDPOINT (OpenRouter, OpenCode Zen Go).
  • Nhận diện luồng suy luận DeepSeek (trích xuất reasoning_content làm dự phòng khi nội dung text trống).
  • Giới hạn max_tokens an toàn tránh lỗi Quota.

Chatbot Baseline vs ReAct Agent

06 / Thử nghiệm
Kịch Bản Thử Nghiệm Kết Quả Chatbot Baseline Kết Quả ReAct Agent Người Chiến Thắng Giải Thích Chi Tiết

Q2: Tích hợp và lưu trữ cơ sở dữ liệu Thất bại. Ảo tưởng câu hỏi và không thể lưu. Thành công bốc đề, thiết kế câu hỏi thiếu và lưu vào JSON. ReAct Agent Agent có khả năng lập luận đa bước (System 2) kết nối trực tiếp với công cụ để thay đổi trạng thái database thực tế.

Hướng Đi Tương Lai & Đưa Vào Sản Xuất

07 / Phát triển
An Toàn & Bảo Mật
  • Input Sanitization: Làm sạch câu lệnh của giáo viên để phòng chống tấn công Prompt Injection làm sai lệch database câu hỏi.
  • Safety Guardrails: Thiết lập bộ lọc nội dung sư phạm nhạy cảm trước khi lưu câu hỏi mới.
Nâng Cấp Hệ Thống
  • Vector Database: Thay thế file JSON bằng VectorDB (ChromaDB/Pinecone) để hỗ trợ tìm kiếm ngữ nghĩa thông minh (Semantic Search).
  • Multi-Agent (LangGraph): Phân rã thành Agent Thiết kế đề và Agent Giáo viên Phản biện chất lượng đề trước khi đồng bộ.
